# К каким системам и почему  относятся MongoDB, MSSQL, Cassandra по CAP теореме.

## Свойства CAP теоремы: 

Согласованность - все рабочие узлы содержат одинаковую информацию. Каждое чтение дает  последнюю запись.

Доступность - возможность доступа к кластеру, даже если узел в кластере выходит из строя. Каждый узел (не упавший) всегда успешно выполняет запросы (на чтение и запись)

Устойчивость к распределению - узлы продолжают работать независимо друг от друга, даже если между ними нет связи.

### Разбор БД

|                                          | MongoDB                                                      | MSSQL                                                        | Cassandra                                                    |
| ---------------------------------------- | ------------------------------------------------------------ | ------------------------------------------------------------ | ------------------------------------------------------------ |
| **Согласованность**   (С)                | Обеспечивает согласованность данных в рамках replica sets. Все операции записи направляются на первичный узел (primary). Когда первичный узел недоступен, выбирается новый первичный из вторичных узлов. В этот период запись приостанавливается, что гарантирует сохранение согласованности данных по всей сети | Обеспечивает согласованность данных через механизмы транзакций и репликации | Жертвует немедленной согласованностью ради доступности. Данные могут быть несинхронизированы между узлами в момент запроса, но система стремится к конечной согласованности. Для устранения несоответствий используется процесс read repair, который при обнаружении противоречий выбирает последние записанные данные |
| **Доступность** (А)                      | При отказе первичного узла запись приостанавливается до избрания нового первичного, что временно снижает доступность системы. | Стремится обеспечить доступность данных, используя механизмы репликации и отработки отказа. В случае сбоя основной базы данных можно переключиться на резервную копию | Обеспечивает высокую доступность: каждый запрос получает ответ, даже если часть узлов недоступна. Каждый узел может обрабатывать запросы на чтение и запись |
| **Устойчивость** **к распределению **(P) | Продолжает работать при сетевых сбоях или отказе отдельных узлов. Вторичные узлы могут временно взять на себя роль первичных, обеспечивая отказоустойчивость | Ориентирована на централизованное управление данными и не рассчитана на работу в условиях полного разрыва связи между узлами | Продолжает работать при сетевых сбоях или отказе узлов. Данные распределены по узлам на основе согласованного хеширования ключа партиционирования |

### Сравнение

| БД            | Категория системы | Приоритеты                                  |
| ------------- | ----------------- | ------------------------------------------- |
| **MongoDB**   | CP                | Согласованность и устойчивость к разделению |
| **MSSQL**     | CA                | Согласованность и доступность               |
| **Cassandra** | AP                | Доступность и устойчивость к разделению     |
